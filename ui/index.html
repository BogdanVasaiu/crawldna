Classic crawl plus the reveal engine (tabs, accordions, “load more” are still clicked, picked by heuristics). The task has no role here — it speaks to the AI, and there is none: the task boxes disappear and files are named from the site. Pages are kept whole and every in-scope link is followed — on a big site that can mean more pages and time, not less. Contain it with Max pages or Include/Exclude.

Any OpenAI-compatible API works — OpenAI, OpenRouter, Groq, Together, or a local server (llama.cpp, LM Studio, vLLM). Pick the model below, or type its id if the list is empty.

Advanced options scope · speed · focus — the defaults are fine
Browser
How the page is opened. auto uses a real browser — needed to click things like tabs or “next month”. never is a quick text-only grab that skips clicking. Keep auto.
Concurrency
How many pages load at once. Higher is faster on big sites; no effect on a single page.
Max pages
Stop after this many pages. 0 = no limit.
Focus on task
How strictly to stay on the task. 0 = off (default — never drops a page). Raise it (e.g. 0.5) to skip links that don't match your task words: faster and more on-topic, but it may miss a page with vague link text. It reads your task, not the site, so it works on any site without rules.
Include
Only follow links whose URL matches this regex.
Exclude
Skip links whose URL matches this regex (wins over Include).
Delay per host
Politeness: minimum milliseconds between requests to the same host (parallel workers queue behind each other there). 0 = off. Slower but gentler — useful on small sites or when a site rate-limits.
Respect robots.txt
Skips URLs the site's robots.txt disallows — each skip shows up as a warning, never silently — and honours its Crawl-delay. Off by default: like wget, crawldna is user-directed.
Embedding model
Optional, from the same provider as the main model (e.g. nomic-embed-text on Ollama — ollama pull nomic-embed-text — or text-embedding-3-small on OpenAI). Makes the task→link relevance semantic: a task in one language ranks pages in another, and synonyms count. It only orders (and feeds “Focus on task” if you raise it) — it never drops a page by itself. Leave it on “none” for the plain word-matching scorer. Hidden with “Crawl without AI”: embeddings are model calls too.

Regex examples: Include /docs/ stays in that section · Exclude /blog/ or \.pdf$ skips those · a|b = a or b · \. = a literal dot. Matching is “contains” by default — leave both empty to follow everything in scope.
For a single page (e.g. a calendar), the defaults are all you need.

Every run is saved automatically — one verbatim .md per link. Reshape it into tables, splits or filtered subsets later, from the saved run.
